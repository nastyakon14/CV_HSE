Вариант A: **система стабилизации камеры**

# Описание проекта

В данном проекте реализована  система **стабилизации видео**, основанная на оценке движения камеры с помощью **оптического потока**.

Были реализованы и сравнены два метода:

* **Lucas–Kanade (LK)** — разреженный оптический поток
* **Farnebäck** — плотный оптический поток

Система выполняет:

1. вычисление движения камеры между кадрами,
2. построение траектории камеры,
3. сглаживание траектории,
4. компенсацию движения (warp),
5. формирование стабилизированного видео.


# Pipeline алгоритма

Алгоритм стабилизации состоит из следующих этапов:

### 1. Чтение видео

Из видео извлекаются кадры и метаданные:

* количество кадров
* размер кадра
* FPS

### 2. Вычисление оптического потока

Для каждой пары соседних кадров вычисляется **матрица аффинного преобразования**, описывающая движение камеры:

* сдвиг по X
* сдвиг по Y
* угол поворота

Используются два метода:

#### Lucas–Kanade

* вычисляет движение **ключевых точек**
* более устойчив к шуму


#### Farnebäck

* вычисляет **плотный оптический поток**
* чувствителен к локальным движениям объектов


### 3. Построение траектории камеры

Движение камеры определяется накоплением трансформаций:

```
trajectory = cumulative_sum(transforms)
```

Получается траектория движения камеры во времени.


### 4. Сглаживание траектории

Чтобы убрать дрожание камеры, траектория сглаживается с помощью **скользящего среднего**:

```
smoothed_trajectory = smooth(trajectory)
```

После этого вычисляется разница между исходной и сглаженной траекторией.


### 5. Компенсация движения

Каждый кадр преобразуется аффинным преобразованием:

```
frame_stabilized = warpAffine(frame, transform)
```

Это компенсирует дрожание камеры.


### 6. Генерация результатов

Алгоритм сохраняет:

* стабилизированное видео
* видео сравнения (Original | Stabilized)
* графики траектории камеры

# Сравнение методов

| Метод        | Особенности                                    | Результат              |
| ------------ | ---------------------------------------------- | ---------------------- |
| Lucas–Kanade | Разреженный поток, отслеживание ключевых точек | Лучше стабилизирует    |
| Farnebäck    | Плотный поток по всем пикселям                 | Более шумный результат |


# Анализ ошибок

Некоторые участки стабилизируются хуже. Возможные причины:


###  Motion blur

При сильном размытии движения оптический поток определяется менее точно.

### Недостаток текстуры

Однородные области (небо, стены) не содержат достаточного количества признаков для корректной оценки движения.

### Ограниченность глобальной модели

Используется **глобальная аффинная модель**, которая не всегда корректно описывает сложное движение камеры.


# Результаты

В результате работы алгоритма получены:

* стабилизированное видео
* видео сравнения
* графики траектории движения камеры

По визуальной оценке **метод Lucas–Kanade показал лучшее качество стабилизации**, чем Farnebäck.

Стабилизированное видео с использованием LK имеет:

* меньше дрожания
* более плавное движение камеры


# Выводы

В ходе работы была реализована система стабилизации видео на основе оптического потока.

Были протестированы два метода оценки движения:

* Lucas–Kanade
* Farnebäck

Эксперименты показали, что **Lucas–Kanade обеспечивает более качественную стабилизацию видео** для конкретного видео.
Это связано с тем, что метод отслеживает устойчивые **ключевые точки**, что позволяет более точно оценивать глобальное движение камеры.

Метод Farnebäck вычисляет плотный поток по всем пикселям, из-за чего он сильнее подвержен влиянию:

* движущихся объектов,
* шума,
* локальных изменений изображения.

В результате стабилизация с использованием Farnebäck оказалась менее стабильной.

Таким образом, для задачи **глобальной стабилизации камеры** более подходящим оказался метод **Lucas–Kanade**.
